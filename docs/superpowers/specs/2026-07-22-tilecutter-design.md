## 1. Project Goal

Build a single-window desktop tool for editing Godot tileset sprite sheets.

The tool must support:

- Importing multiple source tileset images as a material library.
- Grid-based tile selection and copying from source images.
- Pixel-level editing of a target tileset canvas.
- Exporting the final canvas as a PNG and a Godot `TileSet` resource (`.tres`).

---

## 2. Context

The project directory `pic_tile_cut/raw_source/` contains 15 PNG images generated by Gemini.

- Dimensions: `1408 × 768` pixels, all RGBA.
- Common grid fits: `88 × 48` tiles at 16×16, or `22 × 12` tiles at 64×64.
- Source backgrounds use a checkerboard gray color (e.g. `RGBA(127,127,127,255)`), not actual transparency, so a background-removal feature is required.

---

## 3. Requirements

| Requirement | Detail |
|-------------|--------|
| Edit depth | Full pixel editing: brush, eraser, eyedropper, selection tools. |
| Tech stack | Python 3.13 + PyQt5 + PIL + numpy. |
| Export | PNG tileset + Godot `.tres` TileSet resource. |
| Source material | Multiple source images can be opened at the same time. |
| Target canvas | Start from empty canvas; add/remove rows and columns at any time. |
| Undo/redo | Required, implemented with Command pattern. |
| Selection tools | Rectangle select, magic-wand select (contiguous, tolerance-based). |
| Background removal | Global color-to-transparent replacement + magic-wand fine removal. |
| Grid | Any integer tile size; source and target grids must match for copy/paste; support offset and gutter. |
| UI layout | Single main window; top toolbar + options bar like Photoshop/Word; left source library; center canvas; bottom status bar. |
| Snapping | Copy/paste is strictly grid-snapped. |

### Core classes

| Class | Responsibility |
|-------|----------------|
| `MainWindow` | Main window, menus, toolbars, layout, status synchronization. |
| `SourceLibraryWidget` | Left panel: multi-tab source images, grid overlay, tile selection, copy. |
| `CanvasWidget` | Center target canvas: grid rendering, tile rendering, selection overlay, paste preview, mouse handling. |
| `ProjectModel` | Data model: target `QImage`, tile size, offset/gutter, source image list, current selection. |
| `Tool` (abstract) | Base interface for tools: `on_mouse_press/move/release`, tool properties. |
| `BrushTool`, `EraserTool`, `EyedropperTool`, `RectSelectTool`, `MagicWandTool` | Concrete tools. |
| `Command` (abstract) | Undo/redo command interface: `do()` / `undo()`. |
| `GodotExporter` | Exports PNG and `.tres` with correct `texture_region_size`, `separation`, `texture_offset`. |

## 7. Data Model

```python
class ProjectModel:
    image: QImage          # target canvas, RGBA
    tile_width: int
    tile_height: int
    offset_x: int
    offset_y: int
    gutter_x: int
    gutter_y: int
    cols: int
    rows: int
    sources: list[SourceImage]
    selection: SelectionMask | None
    active_tool: Tool
```

Canvas pixel size:

```
width  = offset_x + cols * tile_width  + (cols - 1) * gutter_x
height = offset_y + rows * tile_height + (rows - 1) * gutter_y
```

Adding/removing rows or columns resizes the `QImage` and preserves existing pixels.

## 8. Tools

| Tool | Interaction | Properties |
|------|-------------|------------|
| Brush | Left-drag paints pixels. | Size, hardness, opacity, color. |
| Eraser | Left-drag makes pixels transparent. | Size, opacity. |
| Eyedropper | Click picks color. | - |
| Rectangle Select | Drag creates rectangular selection. | - |
| Magic Wand | Click selects contiguous similar-color region. | Tolerance, contiguous/on-canvas. |
| Color-to-Transparent | Replaces a target color (+ tolerance) with transparent. | Color, tolerance. |

Selection actions:

- `Delete`: clear selected pixels to transparent.
- `Ctrl+C`: copy selected pixels.
- Apply color-to-transparent within selection.

---

## 9. Copy/Paste Flow

1. In `SourceLibraryWidget`, click or drag to select one or more contiguous tiles.
2. Press `Ctrl+C` or click Copy.
3. If source tile size differs from target tile size, show an error dialog and abort.
4. Switch to `CanvasWidget`; mouse shows a semi-transparent paste preview snapped to the grid.
5. Click to place; `PasteCommand` executes and pushes onto `CommandStack`.
6. `Ctrl+Z` reverts the paste.

---

## 10. Undo/Redo

Command pattern. Every reversible user action becomes a `Command`:

- `DrawCommand`
- `EraseCommand`
- `PasteCommand`
- `ClearCommand`
- `ReplaceColorCommand`
- `ResizeCanvasCommand`

`CommandStack` maintains `undo_stack` and `redo_stack`. `Ctrl+Z` undoes, `Ctrl+Y` redoes.

---

## 11. Export

### PNG export

Save the current target canvas `QImage` as PNG using PIL, preserving the alpha channel.

### Godot `.tres` export

Generate a Godot 4 `TileSet` resource referencing the exported PNG:

```ini
[gd_resource type="TileSet" load_steps=2 format=3]

[ext_resource type="Texture2D" path="res://tileset.png" id="1"]

[resource]
sources/0 = SubResource("TileSetAtlasSource_xxxx")

[sub_resource type="TileSetAtlasSource" id="TileSetAtlasSource_xxxx"]
texture = ExtResource("1")
texture_region_size = Vector2i(16, 16)
separation = Vector2i(0, 0)
texture_offset = Vector2i(0, 0)
```

User-configurable export fields:

- Godot resource path for the PNG.
- Tile size.
- Separation (gutter).
- Texture offset.

## 12. Edge Cases

| Case | Handling |
|------|----------|
| Source tile size ≠ target tile size | Error dialog; copy/paste blocked. |
| Paste exceeds canvas bounds | Clip to canvas bounds in the first version. |
| Background removal leaves fully transparent tiles | Preserve alpha; PNG saved normally. |
| Closing with unsaved changes | Show save confirmation dialog. |
| Opening non-PNG files | Reject with a message box. |

---

## 13. Non-Functional Requirements

- **Performance:** Canvas zoom and pan must remain responsive at least up to 4× zoom on a 1408×768 image.
- **Portability:** Code runs on Windows with the existing Python/PyQt5 environment. Avoid OS-specific dependencies beyond PyQt5.
- **Extensibility:** Tool and Command systems are open for adding new tools/commands without changing core widgets.

---

## 14. Out of Scope (for first version)

- Layers.
- Animation preview.
- Tile auto-tiling / terrain generation.
- Non-rectangular tile shapes (isometric, hex).
- Import/export of project files beyond PNG + `.tres`.
