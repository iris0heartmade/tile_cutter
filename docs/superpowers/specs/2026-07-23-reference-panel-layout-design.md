# 参考图区域布局调整设计

日期：2026-07-23

## 目标

将 TileCutter 启动后的整体窗口与左右布局调整为接近 `合适大小截图.jpg` 的视觉效果。截图中的网格已按 4x 显示缩放，因此本次调整针对屏幕显示区域，不改变原始瓦片像素尺寸或默认 4x 缩放逻辑。

## 视觉基准

- 目标窗口比例约为 `2200×1266`。
- 左侧参考图区域初始宽度约为 `800 px`。
- 右侧工作区初始宽度约为 `1400 px`。
- 左右区域比例约为 `1:1.75`，不再使用当前约 `1:2` 的固定初始比例。
- 参考图仍以 4x 显示，截图中约可见 4.5 个放大后的网格宽、3.5 个放大后的网格高。

## 实现范围

1. 调整 `MainWindow` 默认窗口尺寸。
2. 调整主水平 `QSplitter` 的初始尺寸与拉伸策略，使参考区和工作区启动时接近目标比例。
3. 为参考区设置合理的最小宽度，避免窗口初始化或用户缩放时参考图区域过窄。
4. 保持参考图列表、控制按钮、滚动预览、网格配置、同步/非同步模式、拖拽和选区逻辑不变。
5. 确保预览滚动区域继续使用剩余垂直空间，在放大后的窗口中自然扩展。
6. 更新中文 `changelog.txt`，记录本次布局调整。

## 非目标

- 不修改瓦片原始尺寸，默认仍为 `48×48`。
- 不修改参考图默认缩放倍率，默认仍为 `4x`。
- 不改变复制、粘贴、画布编辑和导出行为。
- 不强制固定窗口不可缩放，用户仍可拖动分隔条调整左右比例。

## 验收标准

- 启动应用后，窗口整体比例和截图接近。
- 左侧参考图面板初始宽度接近截图，参考图预览区域明显舒展。
- 右侧工作区仍保留足够编辑面积，并可正常滚动。
- 用户拖动分隔条后，左右区域可正常调整。
- 现有自动化测试通过。
